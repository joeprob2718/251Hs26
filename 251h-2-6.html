2.6: The Runge-Kutta Method

Error Orders of Numerical Methods

Using Taylor approximations, one can show that the maximum possible error (difference between exact and approximate values) for Euler's method is proportional to the step size \(\Delta x\):

\[ |y(x_n) - y_n| = \mathcal{O}(\Delta x). \]

Recall that \(\Delta x = \dfrac{x_n - x_0}{n}\) is the step size when taking \(n\) steps from \(x_0\) to \(x_n\). Therefore, by doubling the number of steps you can expect to become twice as accurate with Euler's method.

The improved Euler method is second order:

\[ |y(x_n) - y_n| = \mathcal{O}\!\left((\Delta x)^2\right). \]

By doubling the number of steps you can expect to become four times as accurate.

One can construct algorithms of any desired order, gaining accuracy without requiring extremely small values of \(\Delta x\). This matters because making \(\Delta x\) smaller requires more steps to get from \(y(x_0)\) to \(y(x_{\text{final}})\).

However, increasing the order too much requires more computations per step (improved Euler needs two function evaluations per step, while regular Euler needs one). The sweet spot is around fourth order — the most commonly used algorithms are the fourth-order Runge-Kutta (RK4) or a hybrid fourth/fifth-order Dormand–Prince method that implements variable step sizes.

Recap: Euler and Improved Euler

Let the step size be \(h = \Delta x\). For the IVP \(\dfrac{dy}{dx} = f(x,y),\; y(x_0) = y_0\), recall the recursive definitions for \(n \ge 0\).

Euler's Method
\[ \begin{aligned} x_{n+1} &= x_n + h, \\ k &= f(x_n,\, y_n), \\ y_{n+1} &= y_n + h\,k. \end{aligned} \]
Improved Euler Method
\[ \begin{aligned} x_{n+1} &= x_n + h, \\ k_1 &= f(x_n,\, y_n), \\ u_{n+1} &= y_n + h\,k_1, \qquad \text{(trial step)}\\ k_2 &= f(x_{n+1},\, u_{n+1}), \\ y_{n+1} &= y_n + \tfrac{1}{2}(k_1 + k_2)\,h. \end{aligned} \]

The improved Euler method averages two slopes: \(k_1\) is the slope \(f(x_n, y_n)\) from regular Euler, and \(k_2\) is the slope at the "trial Euler step" \((x_{n+1}, u_{n+1})\). This yields a second-order method instead of first order.

The Runge-Kutta Method (RK4)

The Runge-Kutta method is a fourth-order scheme that uses a weighted average of four slopes.

Runge-Kutta Method

For the IVP \(\dfrac{dy}{dx} = f(x,y),\; y(x_0)=y_0\), define recursively for \(n \ge 0\):

\[ \begin{aligned} x_{n+1} &= x_n + h, \\[6pt] k_1 &= f(x_n,\; y_n), \\[4pt] k_2 &= f\!\left(x_n + \tfrac{1}{2}h,\; y_n + \tfrac{1}{2}h\,k_1\right), \\[4pt] k_3 &= f\!\left(x_n + \tfrac{1}{2}h,\; y_n + \tfrac{1}{2}h\,k_2\right), \\[4pt] k_4 &= f\!\left(x_n + h,\; y_n + h\,k_3\right), \\[6pt] y_{n+1} &= y_n + \tfrac{1}{6}\,h\left(k_1 + 2k_2 + 2k_3 + k_4\right). \end{aligned} \]

Note that \(k_2\) and \(k_3\) are weighted more heavily than \(k_1\) and \(k_4\) in the final average.

Example: Applying RK4

Example

Approximate \(y(0.5)\) using step sizes \(h = 0.25\) and \(h = 0.1\), then compare with the exact solution for:

\[ \frac{dy}{dx} = x - y, \qquad y(0) = 1. \]

The exact solution is \(y = 2e^{-x} + x - 1\).

Even with just two steps (\(h = 0.25\)), the RK4 approximation already agrees with the exact answer to about four decimal places. With five steps (\(h = 0.1\)), we get six-decimal accuracy.

RK4 is Fourth Order

For a fixed final value \(x_{\text{final}} = x_n\), reducing the step size \(\Delta x = h\) yields greater accuracy. The global error satisfies:

\[ |y(x_n) - y_n| = \mathcal{O}\!\left((\Delta x)^4\right). \]

Here "big-\(\mathcal{O}\) notation" means

\[ a = \mathcal{O}\!\left((\Delta x)^4\right) \quad \Longleftrightarrow \quad a \le C\,(\Delta x)^4, \]

where \(C\) is a constant independent of \(\Delta x\). The value of \(C\) depends on the partial derivatives of \(f(x,y)\); higher-order methods require more differentiability of \(f\).

The practical payoff: if you decrease \(\Delta x\) by a factor of 10 (requiring 10× as many steps), the maximum error decreases by a factor of \(10^4 = 10{,}000\). Compare this to Euler's method, where the same 10× increase in computation only reduces error by a factor of 10.

Local Error vs. Global Error

The global (accumulated) error \(|y(x_n) - y_n|\) is built up from local errors \(|y(x_1) - y_1|\) (the error after a single step). Using Taylor expansions, the local errors for each method are:

MethodLocal ErrorGlobal Error
Euler\(\mathcal{O}((\Delta x)^2)\)\(\mathcal{O}(\Delta x)\)
Improved Euler\(\mathcal{O}((\Delta x)^3)\)\(\mathcal{O}((\Delta x)^2)\)
Runge-Kutta (RK4)\(\mathcal{O}((\Delta x)^5)\)\(\mathcal{O}((\Delta x)^4)\)

Since there are \(n = \mathcal{O}\!\left(\frac{1}{\Delta x}\right)\) steps over which local errors accumulate, the global error is one order less than the local error in each case.

Review: Taylor Expansion and Error Bound

From Math 141, the \(n\)th-degree Taylor polynomial for \(f(x)\) centered at \(a\) is:

\[ T_n(x) = f(a) + f'(a)(x-a) + \frac{f''(a)}{2}(x-a)^2 + \cdots + \frac{f^{(n)}(a)}{n!}(x-a)^n. \]

The Taylor remainder (error bound) gives an accuracy guarantee:

\[ |f(x) - T_n(x)| \le \frac{K}{(n+1)!}\,|x - a|^{n+1}, \]

where \(K = \max |f^{(n+1)}|\) on the interval between \(a\) and \(x\). Abbreviating \(x - a = \Delta x\):

\[ |f(x) - T_n(x)| = \mathcal{O}\!\left((\Delta x)^{n+1}\right). \]

Deriving the Euler Local Error

Example

Consider an autonomous DE: \(\dfrac{dy}{dx} = f(y),\; y(x_0) = y_0\). Show that the Euler method local error is \(\mathcal{O}((\Delta x)^2)\).

Taylor-expand the exact solution around \(x_0\) with step size \(h\):

\[ y(x_0 + h) = y(x_0) + y'(x_0)\,h + \tfrac{1}{2}\,y''(x_0)\,h^2 + \mathcal{O}(h^3). \]

Since \(y(x)\) solves the ODE, we have \(y'(x_0) = f(y_0)\), so:

\[ y(x_0 + h) = \underbrace{y_0 + f(y_0)\,h}_{y_1} + \tfrac{1}{2}\,y''(x_0)\,h^2 + \mathcal{O}(h^3). \]

Therefore the local error is:

\[ |y(x_1) - y_1| = \tfrac{1}{2}\,|y''(x_0)|\,h^2 + \mathcal{O}(h^3). \]

To express \(y''(x_0)\) in known terms, use the chain rule on \(y' = f(y)\):

\[ y'' = f'(y)\,y' = f'(y)\,f(y) \quad \Longrightarrow \quad y''(x_0) = f'(y_0)\,f(y_0). \]

So the local error becomes:

\[ |y(x_1) - y_1| = \tfrac{1}{2}\,|f'(y_0)\,f(y_0)|\,h^2 + \mathcal{O}(h^3). \]

Taking the limit confirms the leading-order behavior:

\[ \lim_{h \to 0} \frac{|y(x_1) - y_1|}{h^2} = \tfrac{1}{2}\,|f'(y_0)\,f(y_0)|, \]

confirming the local error for Euler is second order. Since \(n\) is inversely proportional to \(\Delta x\), the accumulated (global) error after \(n\) steps satisfies \(|y_n - y(x_n)| = \mathcal{O}(\Delta x)\).

Error Scaling Examples

Suppose that for a particular ODE, a method with step size \(h = 0.1\) gives global error \(|y(x_n) - y_n| = 5\). What error is expected if we use \(h = 0.001\) instead? The step size shrinks by a factor of 100.

MethodOrderError reduction factorNew expected error
Euler 1st \(100^1 = 100\) \(\frac{5}{100} = 0.05\)
Improved Euler 2nd \(100^2 = 10{,}000\) \(\frac{5}{10{,}000} = 0.0005\)
Runge-Kutta 4th \(100^4 = 10^8\) \(\frac{5}{10^8} = 5 \times 10^{-8}\)

Code Implementations

Below are Python implementations for each method, set up to solve \(y' = F(x,y)\) with \(x_0 = 0\), \(y_0 = 1\), \(x_{\text{final}} = 1\), and \(N = 10\) subdivisions. Edit the function F and parameters for different problems.

Euler's Method

# Program EULER
N = 10                  # Number of steps
X = 0.0                 # Initial x
Y = 1.0                 # Initial y
X1 = 1.0                # Final x
H = (X1 - X) / N       # Step size

for I in range(N):
    F = X + Y           # Function value f(x, y)
    Y = Y + H * F       # Euler iteration
    X = X + H           # New x
    print(X, Y)         # Display results

Improved Euler Method

# Program IMPEULER
def F(X, Y): return X + Y

N = 10                  # Number of steps
X = 0.0                 # Initial x
Y = 1.0                 # Initial y
X1 = 1.0                # Final x
H = (X1 - X) / N       # Step size

for I in range(N):
    Y0 = Y              # Save previous y
    K1 = F(X, Y)        # First slope
    Y = Y0 + H * K1     # Predictor
    X = X + H           # New x
    K2 = F(X, Y)        # Second slope
    K = (K1 + K2) / 2   # Average slope
    Y = Y0 + H * K      # Corrector
    print(X, Y)         # Display results

Runge-Kutta Method (RK4)

# Program RK
def F(X, Y): return X + Y

N = 10                  # Number of steps
X = 0.0                 # Initial x
Y = 1.0                 # Initial y
X1 = 1.0                # Final x
H = (X1 - X) / N       # Step size

for I in range(N):
    X0 = X              # Save previous x
    Y0 = Y              # Save previous y
    K1 = F(X, Y)        # First slope
    X = X0 + H / 2      # Midpoint x
    Y = Y0 + H * K1 / 2 # Midpoint predictor
    K2 = F(X, Y)        # Second slope
    Y = Y0 + H * K2 / 2 # Midpoint predictor
    K3 = F(X, Y)        # Third slope
    X = X0 + H          # New x
    Y = Y0 + H * K3     # Endpoint predictor
    K4 = F(X, Y)        # Fourth slope
    K = (K1 + 2*K2 + 2*K3 + K4) / 6  # Average slope
    Y = Y0 + H * K      # Corrector
    print(X, Y)         # Display results

Summary

Key Takeaways

The Runge-Kutta method (RK4) computes four slopes per step and takes their weighted average to achieve fourth-order accuracy: the global error scales as \(\mathcal{O}((\Delta x)^4)\). This is the standard workhorse for numerical ODE solving — you don't need to memorize the formulas or compute them by hand, but you should understand why higher-order methods are more efficient and how error scales with step size.
