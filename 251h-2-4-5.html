Euler's Method, Improved Euler, and Runge–Kutta

Why Numerical Methods?

Numerical methods are an important tool in analyzing differential equations. We may want more quantitative information than is available through geometric methods (slope fields, phase lines for autonomous equations), or we may want to explore systems to get a feel for their behavior.

The methods discussed here apply to first-order DEs, but they extend naturally to first-order systems — a form into which all ODEs of any order can be written.

Euler's Method (Section 2.4)

The Idea

Consider the initial value problem

\[ \frac{dy}{dx} = f(x,y), \qquad y(x_0) = y_0. \]

We know both \(y(x_0)\) and \(y'(x_0) = f(x_0, y_0)\). Using the linear (tangent line) approximation:

\[ y(x_0 + \Delta x) \approx y_0 + f(x_0, y_0)\,\Delta x, \]

provided \(\Delta x\) is small. Define \(x_1 := x_0 + \Delta x\) and \(y_1 := y_0 + f(x_0, y_0)\,\Delta x\). Then repeat from \((x_1, y_1)\) to approximate the value at \(x_2 = x_1 + \Delta x\):

\[ y(x_2) \approx y_2 := y_1 + f(x_1, y_1)\,\Delta x. \]

Continue this process indefinitely. This is Euler's method.

Algorithm — Euler's Method

Choose a step size \(\Delta x > 0\). Recursively define for \(n \ge 0\):

\[ \begin{aligned} x_{n+1} &= x_n + \Delta x, \\[4pt] y_{n+1} &= y_n + f(x_n, y_n)\,\Delta x. \end{aligned} \]

Then \(y_n \approx y(x_n)\) is a reasonable approximation to the exact solution at \(x_n\).

Geometrically, each step follows the tangent line at the current point \((x_n, y_n)\) for one step of width \(\Delta x\). The local error is the discrepancy between the tangent line approximation and the true solution curve after one step.

Alternative notation Euler's method can also be written as: \[ k_1 = f(x_n, y_n), \qquad y_{n+1} = y_n + k_1\,\Delta x. \] This notation will be useful when comparing with higher-order methods.

Example — Euler's Method

Problem Approximate \(y(0.5)\) using Euler's method with step sizes \(\Delta x = 0.25\) and \(\Delta x = 0.1\), for the IVP \[ \frac{dy}{dx} = x - y, \qquad y(0) = 1. \] Compare with the exact solution \(y = 2e^{-x} + x - 1\).

The exact value is \(y(0.5) = 2e^{-0.5} + 0.5 - 1 \approx 0.71306\ldots\)

With \(\Delta x = 0.25\) (2 steps)

\(x_0 = 0,\; y_0 = 1.\)

Step 1: \(x_1 = 0.25\).

\[ y_1 = y_0 + f(x_0, y_0)\,\Delta x = 1 + (0 - 1)(0.25) = 0.75. \]

Step 2: \(x_2 = 0.5\).

\[ y_2 = y_1 + f(x_1, y_1)\,\Delta x = 0.75 + (0.25 - 0.75)(0.25) = 0.625. \]
\[ y(0.5) \approx y_2 = 0.625 \qquad\text{(Euler, } \Delta x = 0.25\text{)} \]

With \(\Delta x = 0.1\) (5 steps)

Carrying out 5 steps of Euler's method from \(x_0 = 0\) to \(x_5 = 0.5\):

\[ y(0.5) \approx y_5 = 0.68098 \qquad\text{(Euler, } \Delta x = 0.1\text{)} \]

Smaller step sizes give better approximations — the \(\Delta x = 0.1\) result is closer to the exact value \(0.71306\ldots\)

Improved Euler Method (Section 2.5)

The Idea

Euler's method uses the slope at the beginning of each interval. The improved Euler method (also called Heun's method) improves on this by averaging two slopes: the slope at the beginning and the slope at the end (estimated via a trial Euler step).

Algorithm — Improved Euler Method

Choose a step size \(\Delta x > 0\). Recursively define for \(n \ge 0\):

\[ \begin{aligned} x_{n+1} &= x_n + \Delta x, \\[4pt] k_1 &= f(x_n,\, y_n), \\[4pt] u_{n+1} &= y_n + k_1\,\Delta x, & &\text{(trial step — same as Euler)} \\[4pt] k_2 &= f(x_{n+1},\, u_{n+1}), \\[4pt] y_{n+1} &= y_n + \tfrac{1}{2}(k_1 + k_2)\,\Delta x. & &\text{(real step — uses average slope)} \end{aligned} \]

Then \(y_n \approx y(x_n)\).

Geometrically: compute the slope \(k_1\) at the current point, take a tentative Euler step to get the "trial" point \((x_{n+1}, u_{n+1})\), compute the slope \(k_2\) there, and then take the real step using the average of \(k_1\) and \(k_2\).

Example — Improved Euler Method

Problem Approximate \(y(0.5)\) using the improved Euler method with step sizes \(\Delta x = 0.25\) and \(\Delta x = 0.1\), for the same IVP: \[ \frac{dy}{dx} = x - y, \qquad y(0) = 1. \] The exact solution is \(y = 2e^{-x} + x - 1\), so \(y(0.5) \approx 0.71306\ldots\)

With \(\Delta x = 0.25\) (2 steps)

\[ y(0.5) \approx y_2 = 0.7207\ldots \qquad\text{(Improved Euler, } \Delta x = 0.25\text{)} \]

With \(\Delta x = 0.1\) (5 steps)

\[ y(0.5) \approx y_5 = 0.7142\ldots \qquad\text{(Improved Euler, } \Delta x = 0.1\text{)} \]

Both results are much closer to the exact value than the corresponding Euler method results with the same step sizes.

Error Analysis and Order of Accuracy

Using Taylor approximations, one can show the following bounds on the global error (the difference between the exact and approximate values):

Euler's Method — First Order

\[ |y(x_n) - y_n| = \mathcal{O}(\Delta x). \]

The error is proportional to the step size. Doubling the number of steps (halving \(\Delta x\)) makes the approximation roughly twice as accurate.

Improved Euler's Method — Second Order

\[ |y(x_n) - y_n| = \mathcal{O}\!\left((\Delta x)^2\right). \]

The error is proportional to the square of the step size. Doubling the number of steps makes the approximation roughly four times as accurate.

Big-\(\mathcal{O}\) notation The statement \(y = \mathcal{O}(x)\) means "the size of \(y\) is at most proportional to \(|x|\)," i.e., there exists a constant \(C\) (independent of \(x\)) such that \(|y| \le C|x|\) for \(|x|\) sufficiently small.

The Runge–Kutta Method (Section 2.6)

One can construct numerical algorithms of any desired order, allowing greater accuracy without requiring very small values of \(\Delta x\). This matters because making \(\Delta x\) smaller requires more steps to get from \(y(x_0)\) to \(y(x_f)\).

However, increasing the order requires more function evaluations per step (Euler needs 1, improved Euler needs 2). The practical sweet spot is around fourth order.

Algorithm — Classical Fourth-Order Runge–Kutta (RK4)

For \(n \ge 0\):

\[ \begin{aligned} k_1 &= f(x_n,\, y_n), \\[4pt] k_2 &= f\!\left(x_n + \tfrac{\Delta x}{2},\, y_n + k_1\,\tfrac{\Delta x}{2}\right), \\[4pt] k_3 &= f\!\left(x_n + \tfrac{\Delta x}{2},\, y_n + k_2\,\tfrac{\Delta x}{2}\right), \\[4pt] k_4 &= f\!\left(x_n + \Delta x,\, y_n + k_3\,\Delta x\right), \\[6pt] y_{n+1} &= y_n + \tfrac{1}{6}\!\left(k_1 + 2k_2 + 2k_3 + k_4\right)\Delta x. \end{aligned} \]

This is a fourth-order method: \(|y(x_n) - y_n| = \mathcal{O}\!\left((\Delta x)^4\right)\).

Doubling the number of steps with RK4 makes the approximation roughly 16 times as accurate. It requires 4 function evaluations per step, which is an excellent trade-off for the accuracy gained.

The most commonly used algorithms in practice are the fourth-order Runge–Kutta (RK4) or a hybrid fourth/fifth order Dormand–Prince method that implements variable step sizes for efficient, accurate calculations. In MATLAB, this is the ode45 solver.
